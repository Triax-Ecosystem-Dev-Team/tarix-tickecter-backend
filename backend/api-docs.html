Tarix Ticketer API Documentation

Complete guide to integrating with the Tarix Ticketer backend services. Base URL: http://localhost:5000/api

Authentication

We use JWT-based authentication. Pass the token in the Authorization header for all private routes.

Authorization: Bearer

Dispatch Monitoring

GET /dispatch/fleet-status Private

Fetch real-time occupancy and status for all active and scheduled trips. Uses dynamic Manufacturer + Model naming logic.

Access Role

Admin, Ticketer

GET /dispatch/sales-stats Private

Fetch daily revenue overview grouped by payment method (Cash, Transfer, Card).

Success Response (200)
{
  "success": true,
  "data": {
    "totalTickets": 45,
    "totalRevenue": 850000,
    "breakdown": [
      { "method": "Cash", "count": 20, "amount": 350000 },
      { "method": "Transfer", "count": 15, "amount": 300000 },
      { "method": "Card", "count": 10, "amount": 200000 }
    ]
  }
}
PATCH /dispatch/trips/:id/status Private

Manually update a trip's lifecycle status.

Valid Enums
Scheduled, Active, InTransit, Completed, Delayed, Cancelled

Auth Endpoints

POST /auth/register Public

Register a new user account.

POST /auth/login Public

Login and get an access token.

Request Body
{
  "email": "admin@tarix.com",
  "password": "password123"
}

Trips Management

GET /trips Public

Search and list available trips. Returns professional branding and real vehicle capacity.

Bookings

POST /bookings Private

Create a booking. Note: paymentMethod must be one of Cash, Transfer, or Card.

User Profile & Security

PATCH /users/profile Private

Update user profile details. This endpoint supports avatar uploads using multipart/form-data.

Form Data Fields
Field Type Description
name String User's full name.
phone String Contact phone number.
avatar File Image file (JPG/PNG/WEBP). Uploaded to Supabase.
PATCH /users/security/password Private

Update account password. Requires current password verification.

DevOps Checklist

  • Supabase Storage: Public URL generated for profiles bucket.
  • Prisma 7: Manage connections in prisma.config.ts.
  • CORS: Ensure origin matches your frontend environment.
